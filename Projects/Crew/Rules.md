# Rules

## Принцип: одна шляпа на голове

Текущая роль записана в [[Mode]]. Пока она не сменена — работаю только в рамках этой роли.

## Handoff

PM создаёт `Ticket.md` → Backend берёт → QA проверяет → Tech Lead ревьюит.

Backend НЕ может менять scope тикета. Если понял, что нужно менять `Schema.md` — это блокер. Записать в [[Interrupts]], передать Tech Lead.

## Что нельзя делать в ролях

| Роль | Запрет |
|------|--------|
| PM | Писать код во время "планирования". Придумывать фичи без acceptance criteria. |
| Backend | Менять `Schema.md`. Добавлять поля. Писать end-to-end без QA-шага. |
| QA | Чинить баги. Менять код. |
| Tech Lead | Писать новую функциональность под текущий тикет. |
| DevOps/UX | Менять логику скриптов. |

## Anti-patterns

- **«А я сейчас быстро поправлю»** — Backend берёт QA-шную работу. Результат: баги не замечены, рефакторинг растянулся.
- **«Нет времени на PM, сразу в код»** — превращаешься в разработчика без ТЗ.
- **«Я и PM, и Tech Lead одновременно»** — нет. PM закрывает scope. Tech Lead — ревьюит готовое.

---

Краткая версия: **Решай что делать (PM) → Делай (Backend) → Проверь (QA) → Причеши (Tech Lead). Не смешивай.**
