# Автоматический импорт чека по QR (proverkacheka)

## Проблема

Сейчас `food_import_api.js` для каждой позиции чека вызывает `findOrCreateProduct`, который:
1. Fuzzy-поиск по title
2. Саджестер с выбором существующего / созданием нового
3. 3–5 промптов на название, категорию, единицу

При чеке из 20 позиций это ад. Нужно убрать вопросы полностью.

## Что нужно сделать

1. **Маппинг чек↔продукт**: создать `Projects/Кухня/receipt-product-mapping.json`.
   - Ключ: сырой `name` из чека (как есть от API proverkacheka)
   - Значение: путь к `Product.md`
   - При первом импорте нового названия — записываем соответствие.
   - При повторном — ищем сначала в маппинге (точное совпадение ключа).

2. **Fallback fuzzy-поиск**: если в маппинге нет — fuzzy-поиск по `title` и `aliases` существующих Products (тот же `sortByRelevance`, но без саджестера, автоматический выбор top-1 с порогом score ≥ 60).

3. **Автосоздание Product**: если не найдено ни по маппингу, ни по fuzzy:
   - Нормализовать название (убрать ALL CAPS, развернуть аббревиатуры типа КУР, МОЛ, ШТ)
   - Угадать категорию по ключевым словам (молоко → молочка, яйцо → яйца, паста/макароны → крупы и т.д.)
   - `base_unit` = "шт", остальные поля пустые/дефолтные
   - Создать заметку Product.md автоматически
   - Записать в маппинг

4. **Receipt Item создаётся всегда** с привязкой к найденному или созданному Product.

## Что НЕ трогать

- `food_scan.js` — ручное сканирование штрихкода остаётся как есть
- `food_db.js` — ручное добавление чека через промпты остаётся как есть
- `Schema.md` — если понадобится новое поле, это блокер для Tech Lead
