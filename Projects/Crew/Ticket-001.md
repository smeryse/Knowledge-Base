# Автоматический импорт чека по QR (proverkacheka)

## Проблема

Старый `food_import_api.js` для каждой позиции чека вызывал `findOrCreateProduct`, который останавливался на fuzzy-саджестере и 3–5 промптах. При чеке из 20 позиций это ад.

## Решение

1. **Маппинг чек↔продукт**: `Projects/Кухня/receipt-product-mapping.json`.
   - Ключ: сырой `name` из чека
   - Значение: путь к `Product.md`
   - При первом импорте — записываем. При повторном — ищем в маппинге первым делом.

2. **Fallback fuzzy**: если в маппинге нет — `sortByRelevance` по title, автовыбор top-1.

3. **Автосоздание Product**: если не найдено:
   - Название берётся как есть из чека (без нормализации)
   - Категория = заметка `прочее` (создаётся если нет)
   - `base_unit` = "шт", остальное пустое
   - Записывается в маппинг

4. **Receipt Item** создаётся всегда автоматически.
