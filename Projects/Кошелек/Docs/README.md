# Финансовая база

Главная навигация: [[Кошелек/Кошелек|Главная]] · [[Кошелек/Views|Сводки]]

Система учёта личных финансов внутри Obsidian.

Главная цель: снять хаос в финансах, превратить доходы, расходы, бюджеты и накопления в одну связанную базу.

---

## Сущности

### Account

Счёт, карта или наличные.

Ключевые поля:

- `type: account`
- `title`
- `account_type` — карта/наличные/накопительный
- `balance` — текущий баланс
- `currency` — RUB/USD/EUR
- `bank` — название банка
- `cashback_account` — привязанная программа кешбека

### Category

Категория расходов или доходов.

Ключевые поля:

- `type: category`
- `title`
- `category_type` — expense/income
- `budget_limit` — лимит на месяц (руб)
- `icon` — эмодзи для отображения
- `parent_category` — родительская категория (для иерархии)

### Cashback

Программа кешбека банка.

Ключевые поля:

- `type: cashback`
- `bank`
- `account_id` — привязанный счёт
- `categories` — список MCC-категорий с процентом
- `valid_from` / `valid_to` — период действия
- `cashback_currency` — рубли/баллы/мили

### Budget

Месячный бюджет по категориям.

Ключевые поля:

- `type: budget`
- `month_key` — YYYY-MM
- `total_plan` — общий план расходов
- `total_fact` — факт расходов
- `category_budgets` — таблица: категория, план, факт, остаток
- `food_budget` — отдельный лимит на продукты
- `savings_percent` — процент от ЗП в накопления

### Savings Goal

Цель накоплений.

Ключевые поля:

- `type: savings_goal`
- `title`
- `target_amount`
- `saved_amount`
- `monthly_contribution` — сколько откладывать в месяц
- `monthly_percent` — процент от ЗП
- `deadline`
- `progress` — вычисляемое поле

### Recurring Payment

Регулярный платёж.

Ключевые поля:

- `type: recurring`
- `title`
- `category_id`
- `amount`
- `period` — monthly/quarterly/yearly
- `next_due` — следующая дата списания
- `auto_pay` — автоплатёж вкл/выкл

### Monthly Transactions

Транзакции месяца (временная таблица).

Содержит таблицы доходов и расходов за месяц. До автоматизации ввода — основное место для ручного внесения транзакций.

## Как пользоваться

### Быстро добавить счёт

1. Создай новую заметку в `Projects/Кошелек/Accounts/`.
2. Примени шаблон `Templates/Новый счёт.md`.
3. Заполни метаданные: тип, баланс, валюту, банк.

### Быстро добавить категорию

1. Создай новую заметку в `Projects/Кошелек/Categories/`.
2. Примени шаблон `Templates/Новая категория.md`.
3. Укажи тип (expense/income), лимит бюджета, иконку.

### Быстро добавить кешбек-программу

1. Создай новую заметку в `Projects/Кошелек/Cashback/`.
2. Примени шаблон `Templates/Новый кешбек.md`.
3. Укажи банк, привязанный счёт, MCC-категории с процентами, период действия.

### Создать месячный бюджет

1. Создай новую заметку в `Projects/Кошелек/Budgets/`.
2. Примени шаблон `Templates/Новый месяц.md`.
3. Заполни план по категориям, укажи `food_budget` и `savings_percent`.

### Добавить цель накоплений

1. Создай новую заметку в `Projects/Кошелек/Savings/`.
2. Примени шаблон `Templates/Новая цель накоплений.md`.
3. Укажи целевую сумму, процент от ЗП или фиксированный взнос, дедлайн.

### Добавить регулярный платёж

1. Создай новую заметку в `Projects/Кошелек/Recurring/`.
2. Примени шаблон `Templates/Новый регулярный платёж.md`.
3. Укажи категорию, сумму, период, следующую дату, автоплатёж.

### Внести транзакции за месяц

1. Открой `Projects/Кошелек/Monthly/YYYY-MM.md`.
2. Добавь строки в таблицу доходов или расходов.
3. Обнови `Бюджет/YYYY-MM.md` — перенеси факт из транзакций.

---

## Правила модели

- Один счёт = одна заметка в `Accounts/`
- Одна категория = одна заметка в `Categories/`
- Одна программа кешбека = одна заметка в `Cashback/`
- Один месячный бюджет = одна заметка в `Budgets/`
- Одна цель накоплений = одна заметка в `Savings/`
- Один регулярный платёж = одна заметка в `Recurring/`
- Транзакции месяца = одна заметка в `Monthly/`
- Категории могут иметь иерархию через `parent_category`
- Кешбек считается как доход, уменьшает факт расходов
- Баланс счёта обновляется через транзакции, не редактируется напрямую
- `food_budget` связан с `Projects/Кухня/План питания.md`

## Следующие шаги

- вынести `budget_categories`, `budget_income`, `savings_contributions`, `recurring_transactions`, `cashback_categories` из встроенных таблиц в полноценные сущности
- добавить Templater-скрипты для автоматического обновления факта из транзакций
- подготовить resolver-config.json для будущей LLM-классификации транзакций
- продумать механизм импорта банковских выписок
- когда будет авто-импорт — создать `Transactions/` и перенести транзакции из `Monthly/` в отдельные заметки
