# Продуктовая база

Главная навигация: [[../Index|Index]]

Система для быстрых покупок еды внутри Obsidian.

Главная цель: снять муки выбора и превратить чеки, продукты, запасы дома и список покупок в одну связанную базу.

## Структура

Смотри на `[[Schema]]` как на главный документ по архитектуре.

Коротко:

- `Products/`, `Recipes/`, `Stores/`, `Receipts/`, `Receipt Items/`, `Pantry/`, `Cooking Log/`, `Shopping List/` = таблицы
- `Dashboard.md`, `Рецепты.md`, `Готовка.md`, `Что дома.md`, `Что купить.md` = представления

## Сущности

### Product

Карточка товара.

Ключевые поля:

- `type: product`
- `title`
- `barcode`
- `category`
- `brand`
- `store`
- `base_unit`
- `typical_pack_size`
- `typical_pack_unit`
- `perishable`
- `default_shelf_life_days`
- `price`
- `image`

### Store

Магазин, маркетплейс или точка покупки.

### Recipe

Карточка рецепта.

Ключевые поля:

- `type: recipe`
- `title`
- `dish_type`
- `servings`
- `total_time_min`
- `source`
- `products`
- `recipe_status`

### Receipt

Один чек целиком.

### Receipt Item

Одна позиция внутри чека.

Это главная аналитическая сущность: именно по ней можно смотреть историю покупок и цены, если понадобится.

### Pantry Item

Один конкретный домашний запас.

Нужен, чтобы вести остатки и сроки годности по отдельным партиям, а не абстрактно по товару.

### Shopping Item

Пункт списка покупок.

### Cooking Entry

Одна запись о фактическом приготовлении блюда.

Нужна, чтобы фиксировать не просто рецепт как справочник, а конкретный факт готовки и списание остатков из `Pantry`.

## Как пользоваться

### Быстро добавить чек

1. Создай новую заметку в `Projects/Еда/Receipts/`.
2. Применяй шаблон `Projects/Еда/Templates/Новый чек`.
3. Мастер спросит дату, магазин и товары по одному.
4. Если товар не найден, мастер предупредит и создаст новую карточку.
5. Для скоропорта можно сразу завести срок годности в домашнем запасе.

### Быстро добавить товар по штрихкоду

1. Открой любую временную заметку или отдельную страницу-сканер.
2. Примени шаблон `Projects/Еда/Templates/Скан товара`.
3. Вставь или отсканируй штрихкод в prompt.
4. Система сначала ищет товар в локальной базе.
5. Если локально не найдено, она пробует найти товар в интернете по штрихкоду.
6. Если найдено, можно принять найденное название и создать карточку с предзаполненными полями.
7. Если не найдено, система создаст карточку вручную и привяжет к ней штрихкод.

### Локальная модель для нормализации

1. Настройки лежат в `Projects/Еда/resolver-config.json`.
2. По умолчанию ожидается `LM Studio` OpenAI-compatible endpoint `http://127.0.0.1:1234/v1`.
3. Модель по умолчанию: `qwen2.5-3b-instruct`.
4. Resolver собирает сырые кандидаты из интернета и просит локальную модель привести их к строгому JSON под твою схему товара.
5. Если локальная модель недоступна, система просто берёт первый удачный источник без LLM-нормализации.
6. Поддерживаются два провайдера: `lmstudio` и `ollama`.

### OCR чека

1. На desktop можно применить шаблон `Projects/Еда/Templates/OCR чек`.
2. Укажи путь к фото чека в `assets/`.
3. Скрипт прогонит изображение через `tesseract` и вставит черновой текст в заметку.
4. Это вспомогательный режим: лучше использовать его для черновика, а не как полностью автоматический парсер.

### Ручной режим

Можно создавать заметки через шаблоны сущностей в `Projects/Еда/Templates/` и потом редактировать их в `Bases`.

### Добавить рецепт

1. Создай новую заметку в `Projects/Еда/Recipes/`.
2. Примени шаблон `Projects/Еда/Templates/Новый рецепт.md`.
3. Заполни метаданные рецепта и список `products` ссылками на товары.
4. В секции `Ингредиенты` укажи количества, а в `Шаги` сам процесс приготовления.

### Списать продукты по приготовлению

1. Примени шаблон `Projects/Еда/Templates/Приготовить блюдо.md`.
2. Выбери рецепт из `Recipes/`.
3. Укажи, сколько порций реально приготовлено.
4. Скрипт разберёт таблицу `Ингредиенты`, спишет поддерживаемые позиции из `Pantry` и создаст запись в `Cooking Log/`.
5. Для автоматического списания в рецепте должны быть числовые количества и единицы `г`, `кг`, `мл`, `л` или `шт`.

## Правила модели

- Один чек = одна заметка в `Receipts/`
- Одна строка в чеке = одна заметка в `Receipt Items/`
- Один физический запас дома = одна заметка в `Pantry/`
- Один рецепт = одна заметка в `Recipes/`
- Одно приготовление блюда = одна заметка в `Cooking Log/`
- Для скоропорта срок лучше хранить на уровне `Pantry Item`
- Цены и магазины храни в чеках и позициях чека, а не в карточке товара
- В карточке товара `price` можно держать как текущую опорную цену для планирования бюджета
