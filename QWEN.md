## Система баллов (ВАЖНО!)

### Команда: `проставь баллы` или `проставь баллы за [дата]`

**Что делать:**
1. Найти файл `Tasks/Daily/YYYY-MM-DD.md` (текущий день или указанная дата)
2. Прочитать все задачи (`- [ ] ...`)
3. **Глубоко проанализировать контекст** каждой задачи — понять сложность, объём, тип
4. Добавить баллы в формате: `- [ ] Задача` → `- [ ] Задача (+N)`
5. **НЕ менять статус** — `[ ]` остаётся `[ ]`

### Принципы оценки (по контексту, не шаблонно!):
- **Рутина** (постельное, мелкое) → +5–15
- **Учебные** (реферат, лабы, разбор задач) → +15–50
- **Технические** (установка ПО, настройка, прокси, VPN) → +10–25
- **Проекты** (начало/продолжение кода) → +20–50
- **Организация** (структура, индексация) → +10–25
- **Работа/заработок** → по фактической сумме (озон фреш и т.д.)
- **Покупки** → +20–30

### Формат:
```markdown
- [ ] Сменить постельное (+15)
- [ ] Начать реферат (+20)
- [ ] Настроить прокси (+20)
```

### НЕ делать:
- НЕ менять `[ ]` на `[x]`
- НЕ добавлять `✅ дату выполнения`
- НЕ трогать уже выполненные задачи с `[x]`

### Система калибровки:
После команды `проставь баллы` пользователь может дать обратную связь (например "Слишком мало баллов"). Нужно:
- Скорректировать баллы в текущем файле
- Записать скорректированное значение в таблицу ниже для будущего

### Таблица корректировок:
| Задача | Моя оценка | Корректировка пользователя | Итого |
|--------|-----------|---------------------------|-------|
| Приготовить завтрак/обед/ужин на 3 дня | +25 | "Слишком мало" | +50 |
| Проработать структуру VPN в v2rayN (только архитектура) | +25 | "Слишком много, только архитектура" | +15 |
