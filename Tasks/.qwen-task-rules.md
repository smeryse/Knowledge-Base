# Правила простановки баллов

## Команда: `проставь баллы` или `проставь баллы за [дата]`

### Что делать:
1. Найти файл дня в `Tasks/Daily/YYYY-MM-DD.md`
2. Прочитать все задачи (`- [ ]`)
3. **Глубоко проанализировать контекст** каждой задачи — сложность, объём, тип
4. Добавить баллы: `- [ ] Задача` → `- [ ] Задача (+N)`
5. **НЕ менять статус** — `[ ]` остаётся `[ ]`

### Принципы оценки:

| Тип задачи | Диапазон | Примеры |
|------------|----------|---------|
| **Рутина** (постельное, еда, мелкое) | +5 – +15 | "Сменить постельное" (+15), "Закупиться продуктами" (+30) |
| **Учебные** (реферат, разбор задач) | +15 – +50 | "Разбор касд задачи" (+20), "Сдать 4 лабы" (+50) |
| **Технические** (установка ПО, настройка) | +10 – +25 | "Установить Upscayl" (+10), "Прокси тг" (+20) |
| **Проекты** (начало/продолжение кода) | +20 – +50 | "Начать Telegram-бота" (+50) |
| **Организация** (структура, индексация) | +10 – +25 | "Связать папки" (+10), "Поправить расписание" (+15) |
| **Привычки** | +5 – +20 | "Выпить 2л воды" (+10), "Спорт" (+20) |
| **Работа/заработок** | по сумме | "Озон фреш" (+1128р) |

### Оценка:
- **5-15** — мелкое/быстрое
- **15-25** — средняя задача
- **25-50** — серьёзная работа

### Формат добавления:
```markdown
- [ ] Задача (+N)
```

### НЕ делать:
- НЕ менять `[ ]` на `[x]`
- НЕ добавлять дату выполнения `✅`
- НЕ трогать уже выполненные задачи (`[x]`)
