Привет, Ярослав. Вот краткая шпаргалка по плагину Obsidian Charts.

**Базовый блок**
```charts
{
  "type": "bar",
  "data": {
    "labels": ["Янв", "Фев", "Мар"],
    "datasets": [{
      "label": "Выручка",
      "data": [1200, 1500, 900],
      "backgroundColor": "rgba(54, 162, 235, 0.5)",
      "borderColor": "rgba(54, 162, 235, 1)",
      "borderWidth": 1
    }]
  },
  "options": {
    "responsive": true,
    "plugins": {
      "title": { "display": true, "text": "Квартальные показатели" },
      "legend": { "position": "top" }
    }
  }
}
```

**Ключевые параметры**
- `type`: `bar`, `line`, `pie`, `doughnut`, `radar`, `polarArea`, `scatter`
- `data.labels`: метки оси X или сектора
- `data.datasets`: массив наборов. Каждый содержит `label`, `data`, цвета, тип линии
- `options`: управление осями, легендой, подсказками, анимацией

**Полезные опции**
- `scales.y.beginAtZero: true` — фиксация нуля на оси Y
- `plugins.legend.display: false` — скрыть легенду
- `elements.line.tension: 0.4` — сглаживание линий
- `animation: false` — отключить анимацию для стабильного рендера

**Работа с данными**
- Плагин принимает чистый JSON внутри блока ` ```charts `
- Поддерживает вставку CSV через поле `"csv": "строка"` вместо `data`
- Для динамических значений используйте DataviewJS: генерируйте JSON-объект и выводите его в тот же блок
- Даты передавайте в формате `"YYYY-MM-DD"` или `"MM/YYYY"`

**Ограничения**
- Синтаксис соответствует Chart.js v4. Внутри блока не выполняются JS-функции, только статичные значения
- Сложные кастомные скрипты не поддерживаются из-за sandbox-рендеринга
- При изменении данных требуется сохранить заметку или перезапустить просмотр

Документация Chart.js покрывает все доступные поля. Если нужно разобрать конкретный тип графика или связку с Dataview, напиши.
