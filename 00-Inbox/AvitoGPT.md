Коротко и по делу. Для такого проекта у тебя **три крупных блока**:

1. диалоговая логика (LLM → уточняющие вопросы),
    
2. сбор и анализ данных с b\у-площадок,
    
3. ранжирование и принятие решения (какие объявления подходят лучше).
    

---

## 1. Диалоговая часть (нейросеть задаёт вопросы)

Что нужно знать:

### ✅ NLP / LLM

- Как работают **языковые модели** (GPT, LLaMA, Mistral).
    
- Prompt engineering: как формировать запрос модели.
    
- Few-shot / chain-of-thought (на уровне структуры: как подвести модель к вопросу).
    

### ✅ Designing dialog flow

- Машина состояний / decision tree или **динамическая структура вопросов**.
    
- RAG (Retrieval-Augmented Generation) — если будешь подгружать справочную инфу, например "чем Telecaster отличается от Strat".
    

---

## 2. Сбор данных (Авито, Юла)

Нюанс: **Авито и Юла закрывают API и защищают от парсинга**.

Прямой парсинг может нарушить ToS, поэтому:

### ✅ Варианты

- Неофициальные API (например, Avito API для партнеров — можно попробовать получить).
    
- Парсинг через Playwright / Selenium (имитировать браузер).
    
- Парсинг HTML + регулярки + обработка естественного текста объявления.
    

### Что обязательно изучить:

- **Web scraping:** BeautifulSoup / Playwright / Selenium.
    
- **Антибот:** смена user agent, задержки, headless browser.
    
- **Очереди и фоновые задачи:** Celery / Redis (для сборов данных).
    

---

## 3. Оценка и ранжирование объявлений

Тут начинается "магия" проекта.

### Что изучить:

- **Embeddings (векторизация текста)**: для сравнения того, что хочет пользователь, и описания товара.
    
    - Sentence transformers / OpenAI embeddings / text-embedding-3-large
- **Similarity search (поиск похожего):**
    
    - FAISS / Pinecone / ChromaDB
- **Simple scoring model (MVP)**
    
    - Цена vs состояние vs дефекты vs профиль продавца.
- **Regression / классификация**
    
    - Можно сделать модель, которая предсказывает "Скорее всего покупатель будет доволен этим выбором".

Пример формулы (MVP):

```

score = relevance * 0.6 + price_score * 0.3 + seller_score * 0.1

```

## Архитектура MVP

```

User → Chat (LLM) → Сбор требований

                      ↓

                Поиск объявлений (scraper)

                      ↓

      Векторизация описания и сравнение с запросом

                      ↓

           Отбор и ранжирование → Ответ пользователю

```
