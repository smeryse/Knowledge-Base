## Таблица «Команды поиска и их назначение»

| Команда | Что делает | Кратко |
|---------|------------|--------|
| **`which <команда>`** | Находит путь к исполняемому файлу, который будет запущен, если пользователь просто напишет `<команда>` в терминале. | Смотрит только по переменной окружения `PATH`. |
| **`whereis <команда>`** | Ищет не только исполняемый файл, но и исходный код, документацию (man‑страницы) и другие связанные файлы для указанной команды. | Использует заранее подготовленные списки путей (`/usr/bin`, `/bin`, `/sbin`, `/usr/share/man` и т.д.). |
| **`type <команда>`** | Показывает, как Bash будет интерпретировать `<команда>`: это встроенная команда, алиас, функция, исполняемый файл и т.д. | Встроена в Bash/клоу. |
| **`command -v <команда>`** | Аналогично `type`, но возвращает только путь к исполняемому файлу (если он есть). | Подходит для скриптов. |
| **`which -a <команда>`** | Показывает все варианты пути, найденные в `PATH`. | Полезно при наличии нескольких версий одной программы. |
| **`locate <фрагмент_имени>`** | Быстрый поиск файлов по базе данных (обновляется командой `updatedb`). | Может найти файлы, не связанные с командами. |
| **`find / -name <pattern> 2>/dev/null`** | Полноценный поиск по файловой системе в реальном времени. | Медленнее, но точнее. |
| **`whereis -B /usr/bin -M /usr/share/man -b ls`** | Позволяет задать собственные каталоги поиска для исполняемых файлов и man‑страниц. | Гибкая настройка путей. |
| **`which -p <команда>`** (не в GNU) | Ищет только по `PATH`, игнорируя алиасы/функции. | Уточнённый поиск без расширений Bash. |

> **Пример использования:**
>
> ```bash
> $ which ls
> /bin/ls
>
> $ whereis ls
> ls: /bin/ls /usr/share/man/man1/ls.1.gz
> ```

Эта таблица объединяет все команды из исходного файла в единую структуру, сохраняя ключевые сведения о каждой из них.
