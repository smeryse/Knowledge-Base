# Основные модели

## Kimi K2.6 — default model

Использовать для:
- повседневного coding
- Ubuntu/Linux задач
- SSH
- bash scripting
- Docker
- nginx
- systemd
- GitHub workflows
- API integration
- multi-file edits
- refactoring
- debugging
- long agent sessions

### Почему
- лучший balance
- хороший tool use
- стабильный агентный workflow
- быстрый
- хорошо держит контекст
- аккуратнее DeepSeek

### Не идеален для
- extremely complex reasoning
- critical architecture decisions
- very dangerous infra changes

---

# DeepSeek V4 Pro — heavy coding / infra

Использовать для:
- сложного debugging
- networking
- low-level Linux
- production infra analysis
- performance optimization
- backend internals
- Rust/Go/C++ heavy tasks
- reverse engineering
- massive code analysis

### Почему
- очень сильный raw coding
- мощный technical reasoning
- хорошо пишет bash
- силен в infra

### Минусы
- aggressive edits
- может ломать codebase
- хуже агентная дисциплина
- иногда слишком уверен в неправильных решениях

### Правило
НЕ давать полный autonomous control без проверки diff.

---

# GLM 5 — reasoning / planning

Использовать для:
- архитектуры
- planning
- multi-step reasoning
- сложных agent chains
- long-context analysis
- research
- проектирования систем
- CI/CD planning
- migration planning

### Почему
- хороший reasoning
- длинный контекст
- неплохая агентность

### Минусы
- менее проверенный ecosystem
- weaker coding consistency vs Kimi/DeepSeek

# Routing Rules

## По умолчанию
→ Kimi K2.6

## Если:
- сложный backend
- networking
- kernel/system issues
- performance
- infra debugging

→ DeepSeek V4 Pro

## Если:
- architecture
- planning
- long reasoning
- migration strategy

→ GLM 5

## Если:
- cheap/simple task
- formatting/docs

→ GLM 4.5 Air

---

# DevOps Safety Rules

## Всегда:
- проверять diff
- использовать git
- использовать isolated worktrees
- запускать dry-run где возможно
- не давать агенту root без необходимости
- проверять:
  - nginx configs
  - firewall rules
  - docker compose
  - systemd units
  - ssh configs

---

# Recommended Workflow

## Серверы
1. Agent анализирует
2. Генерирует plan
3. Review diff
4. Dry-run
5. Apply
6. Validate
7. Commit

# Dangerous Tasks

## НЕ давать fully autonomous execution для:
- firewall
- iptables/nftables
- sshd_config
- production database migration
- rm -rf operations
- recursive chmod/chown
- DNS changes
- system package cleanup
