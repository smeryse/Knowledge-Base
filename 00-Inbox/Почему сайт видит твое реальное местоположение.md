**Сайт не «видит» вас напрямую — он получает данные, которые вы (или браузер) ему передаёте.** Если ваш реальный регион «утек», значит, один из каналов связи не закрыт.

Даже с включенным VPN существует 7 основных векторов утечки. Вот чеклист для диагностики и устранения на Linux.

---

### 🔍 1. Быстрая самодиагностика

Выполните эти проверки **в том же браузере**, где открыли Wikidot:

```bash
# 1. Проверка реального IP (кто вы для внешнего мира?)
curl ifconfig.me
curl ipv4.icanhazip.com

# 2. Проверка DNS-резолвера (через чьи сервера идут запросы?)
curl https://dnsleaktest.com/test.json 2>/dev/null | grep -o '"country":"[^"]*"' | head -1

# 3. Проверка заголовков браузера (язык и локаль)
curl -A "Mozilla/5.0" -H "Accept-Language: ru-RU,ru" -s https://httpbin.org/headers | grep -i accept
```

**Онлайн-тесты для комплексной проверки:**
*   [ipleak.net](https://ipleak.net) — показывает IP, DNS, WebRTC и геолокацию.
*   [browserleaks.com](https://browserleaks.com) — детальный анализ отпечатка браузера (канвас, шрифты, время).
*   [whoer.net](https://whoer.net) — оценка анонимности в %.

### 🕵️ 7 основных векторов утечки (от частых к редким)

#### 1. 🌐 DNS-утечка (Самая частая причина)
VPN туннелирует трафик, но запросы к доменным именам (DNS) могут уходить через провайдера. Сайт видит запрос от российского DNS-сервера.

**Проверка:**
```bash
resolvectl status  # systemd-resolved
# или
cat /etc/resolv.conf
```
Если видите там `8.8.8.8` (ваш домашний или локальный) вместо DNS вашего VPN — это утечка.

**Решение (Ubuntu/NetworkManager):**
```bash
# Запретить системе использовать локальные DNS
sudo nmcli connection modify "Ваш_Коннект" ipv4.ignore-auto-dns yes
sudo nmcli connection modify "Ваш_Коннект" ipv4.dns "1.1.1.1 1.0.0.1"
sudo nmcli connection up "Ваш_Коннект"
```
*Или включите "DNS Leak Protection" в настройках VPN-клиента.*

#### 2. 🕐 Timezone и Locale (Отпечаток браузера)
Сайт получает ваше время через JavaScript. Если VPN в США, а время `Europe/Moscow` — это 100% палево.

**Проверка в консоли браузера (F12):**
```javascript
console.log(Intl.DateTimeFormat().resolvedOptions().timeZone);
console.log(navigator.language);
```

**Решение:**
*   **Firefox:** `about:config` → `javascript.use_us_english_locale` → `true` (или использовать расширения для спуфинга, например, *Timezone Spoofing*).
*   **Chrome/Chromium:** Запуск с флагами:
    ```bash
    google-chrome --timezone=America/New_York --lang=en-US
    ```

#### 3. 🗣️ Заголовок Accept-Language
Браузер сообщает сайту предпочтительный язык. Строка `ru-RU,ru;q=0.9` выдает русскоязычного пользователя, даже если он зашел с айпи Нидерландов.

**Решение:**
Установите в браузере английский язык интерфейсом по умолчанию или используйте расширения для подмены заголовков (например, *User-Agent Switcher*).

#### 4. 📡 WebRTC Leak
Технология для видеочатов может показать ваш реальный локальный IP, обходя VPN.

**Проверка:** [browserleaks.com/webrtc](https://browserleaks.com/webrtc)

**Решение (Firefox):**
`about:config` → `media.peerconnection.enabled` → `false`.
**Решение (Chrome):** Расширения типа *WebRTC Control*.

#### 5. 🌍 IPv6 Leak
Многие VPN туннелируют только IPv4. Если ваш провайдер поддерживает IPv6, трафик пойдет напрямую через него.

**Проверка:**
```bash
curl -6 ifconfig.me
```
Если команда вернула ваш реальный домашний IPv6 — отключите его в системе.

**Решение (через sysctl):**
```bash
sudo sysctl -w net.ipv6.conf.all.disable_ipv6=1
sudo sysctl -w net.ipv6.conf.default.disable_ipv6=1
```

#### 6. 🗺️ Geolocation API
Сайт может запросить доступ к геолокации (через Wi-Fi точки доступа или GPS). Если вы нажали «Разрешить» в браузере — данные ушли напрямую, минуя VPN.

**Решение:** Всегда блокировать запросы на доступ к местоположению в настройках приватности браузера.

#### 7. 🍪 Cookies и кэш
Если вы зашли на Wikidot *до* включения VPN, у сайта уже есть куки с вашей старой геолокацией.

**Решение:** Всегда используйте режим «Инкогнито» или чистите куки перед сменой локации.

### 🛡️ Профиль для OSINT-работы (Firefox)

Для максимальной анонимности рекомендую использовать **Firefox** с жесткими настройками в `user.js` (или вручную в `about:config`):

```javascript
// Блокировка телеметрии и утечек
user_pref("privacy.resistFingerprinting", true);
user_pref("privacy.trackingprotection.enabled", true);
user_pref("media.peerconnection.enabled", false); // Отключаем WebRTC
user_pref("geo.enabled", false); // Запрет геолокации
user_pref("intl.accept_languages", "en-US, en"); // Язык только английский
user_pref("privacy.spoof_english", 2); // Спуфинг языка
user_pref("network.dns.disablePrefetch", true); // Отключение предзагрузки DNS
```

> **Важно:** Включение `privacy.resistFingerprinting` может «сломать» отображение некоторых сайтов, но для разведки это необходимая жертва.

### ✅ Итоговый чеклист перед заходом на сайт

1.  [ ] Включен VPN, проверен IP через `curl ifconfig.me`.
2.  [ ] Нет утечек DNS (проверено на ipleak.net).
3.  [ ] IPv6 отключен или туннелируется.
4.  [ ] Часовой пояс и язык браузера соответствуют стране выхода VPN.
5.  [ ] Открыт режим Инкогнито (чтобы исключить куки).
6.  [ ] Запросы на геолокацию заблокированы.

Если после этого сайт всё равно определяет Россию — проблема может быть на стороне базы геолокации по IP (иногда российские подсети ошибочно помечены), но это редкость. Скорее всего, один из пунктов выше не выполнен.
