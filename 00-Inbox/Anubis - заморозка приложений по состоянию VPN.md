# Anubis: заморозка приложений по состоянию VPN

## Суть проблемы
- Приложения (банки, маркетплейсы) могут детектировать VPN и отправлять его IP в РКН для блокировки.
- Векторы детекта: `ConnectivityManager`, интерфейс `tun0`, маршруты `/proc/net/route`, SOCKS5 на localhost.
- Рабочий профиль (Island/Insular) скрывает VPN только от `ConnectivityManager`, но не от низкоуровневых проверок.
- Фоновая активность приложений продолжает работать даже в песочнице.

## Решение: полная заморозка через `pm disable-user`
- Отключённое приложение не имеет процесса, не получает интенты, не работает в фоне — физически не может «стучать».
- Команда требует shell-доступа → используется **Shizuku** (ADB-права без root).

## Архитектура Anubis

### Три группы приложений
| Группа | По умолчанию | При запуске из Anubis |
|--------|-------------|----------------------|
| **Без VPN** | Заморожено | Выкл. VPN → разморозить → запустить |
| **Только VPN** | Заморожено | Вкл. VPN → разморозить → запустить |
| **Запуск с VPN** | Активно | Вкл. VPN → запустить (без заморозки) |

### Ключевые механизмы
- **Управление VPN**: через `am broadcast` (widget toggle) или `am start` (Activity) для известных клиентов (v2rayNG, NekoBox, Happ и др.).
- **Определение активного VPN-клиента**: парсинг `dumpsys connectivity` → получение UID → резолв в package name.
- **Выключение любого VPN**: хак через `VpnService` — создаётся «dummy» VPN на долю секунды, система автоматически отзывает предыдущий.
- **Реверс закрытых клиентов**: поиск broadcast-action через декомпиляцию ресурсов APK (jadx) — виджеты не обфусцируются.

## Ограничения и нюансы
- Замороженное приложение не получает push-уведомления.
- При долгой заморозке возможен выход из аккаунта (зависит от приложения).
- Требуется одноразовая настройка Shizuku (ADB/Wireless Debugging).
- «Постоянная VPN» (Always-on) в настройках Android может мешать отключению.
- Некоторые китайские приложения (WeChat и др.) могут блокировать аккаунт при обнаружении Shizuku.

## Заметки
> [!tip] Практическое применение
> Использовать для изоляции «подозрительных» приложений (банки, маркетплейсы) при работе с чувствительными сервисами через VPN. Не забывать про trade-off: безопасность ↔ уведомления.

> [!warning] HyperOS / MIUI
> На кастомных оболочках возможны проблемы с заморозкой. Проверять актуальные релизы на GitHub.
