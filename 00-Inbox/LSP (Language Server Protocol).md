**LSP** — открытый протокол на базе JSON-RPC, который стандартизирует обмен данными между текстовым редактором (клиентом) и языковым сервером, предоставляющим функции «языкового интеллекта»: автодополнение, переход к определению, подсветку ошибок, рефакторинг и др. [[2]]

### Как работает
- Редактор (клиент) отправляет серверу уведомления о действиях пользователя: открыл файл, ввёл символ, выделил диапазон.
- Клиент может запросить у сервера языковую функцию: «покажи подсказки», «найди все ссылки на этот символ».
- Сервер отвечает в формате JSON-RPC: возвращает данные или ошибку. [[2]]
- Протокол не диктует способ транспорта: клиент и сервер могут быть в одном процессе или общаться по сети. [[2]]

### Зачем это нужно
- **До LSP**: поддержка языка «зашивалась» в каждый редактор отдельно. Чтобы добавить одну фичу в 5 редакторов — нужно 5 реализаций.
- **С LSP**: пишется один языковой сервер — и он работает в любом редакторе, который поддерживает протокол (VS Code, Neovim, Sublime, Emacs и др.). [[2]]
- Это снижает порог входа для создания инструментов под новые языки и упрощает поддержку существующих.

### Применительно к Obsidian
Obsidian сам по себе не поддерживает LSP нативно, но есть обходные пути:
- **obsidian-lsp** — LSP-сервер для Markdown-файлов из вашего хранилища Obsidian. Позволяет подключить редакторы типа Neovim и получить: переход по ссылкам, автодополнение тегов/свойств, поиск обратных ссылок, переименование файлов с обновлением всех ссылок. [[11]]
- **Markdown-Oxide** — плагин, использующий LSP для расширения возможностей редактирования. [[15]]
- **obsidian.nvim** — плагин для Neovim, который автоматически запускает LSP-сервер при работе с хранилищем Obsidian. [[18]]

### Коротко
> Если хочешь редактировать заметки Obsidian в продвинутом редакторе (Neovim, VS Code) с автодополнением, навигацией и рефакторингом — LSP позволяет подключить к хранилищу языковой сервер, который «поймёт» структуру твоих заметок и даст удобные инструменты.
