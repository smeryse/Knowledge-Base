**Технический промпт для ChatGPT:**

> «У меня GNOME‑расширение для управления яркостью внешнего монитора через DDC/CI (`gnome-display-brightness-ddcutil`). Система Ubuntu, иногда использую Wayland/Xorg.
> 
> Что уже сделано:
> 
> 1. `ddcutil detect` работает только при прямом доступе к I²C. Первый запуск после cold boot вызывает зависание системы на несколько секунд.
>     
> 2. Временно добавлен пользователь в группу `i2c` (`sudo gpasswd -a $USER i2c`), чтобы получить права на `/dev/i2c-*`.
>     
> 3. Создан кэш detect для GNOME: `ddcutil --brief detect > ~/.cache/ddcutil_detect`. Теперь расширение использует кэш и не зависает.
>     
> 4. Права на I²C отозваны (`sudo gpasswd -d $USER i2c`), расширение продолжает работать через кэш.
>     
> 5. При отключении/подключении монитора кэш остаётся, расширение не рекеширует автоматически.
>     
> 6. Для проверки состояния можно запускать `ddcutil detect` или `ddcutil environment`.
>     
> 
> Задача: проверить корректность работы расширения, обновление кэша при подключении/отключении монитора, безопасное использование без постоянного доступа к I²C. Подскажи пошагово, как восстановить или отладить работу.»
