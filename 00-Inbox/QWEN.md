# QWEN.md — Контекст для AI-ассистентов

## 📌 Обзор проекта

Это **персональная база знаний (Knowledge Base)** на основе **Obsidian**, содержащая заметки, задачи, техническую документацию и личную информацию. База организована по тематическим категориям с использованием системы тегов и метаданных.

**Тип проекта:** Non-Code (документация/заметки)

## ✍️ Формат заметок

### Frontmatter (YAML)

Большинство заметок начинаются с frontmatter:

```yaml
---
tags:
  - категория
  - подкатегория
banner: "URL_или_путь_к_изображению"
cssclasses:
  - custom-class
aliases:
  - Альтернативное имя
---
```

### Синтаксис

- **Внутренние ссылки:** `[[Название заметки]]`
- **Внешние ссылки:** `[текст](url)`
- **Изображения:** `![[image.png]]` или `![](url)`
- **Задачи:** `- [ ] задача` / `- [x] выполнено`
- **Dataview:** Блоки кода с запросами к базе

## 🔧 Технические особенности

### Obsidian-конфигурация

- **Плагины:** Dataview, Charts, Kanban, Projects, Banners, Tag Wrangler, Admonitions
- **Темы:** Кастомные CSS-сниппеты в `.obsidian/snippets/`
- **Настройки:** В `.obsidian/` (app.json, appearance.json, workspace.json)

### Dataview

База активно использует **Dataview** для динамических списков:

```dataview
TABLE file.name, file.tags
FROM #linux
SORT file.mtime DESC
```

## 🎯 Правила работы с базой

1. **Всегда начинайте с `00-AI-INSTRUCTIONS.md`** — там основные правила
2. **Проверяйте `00-task.md`** перед работой — актуальные задачи
3. **Не удаляйте файлы** без явного запроса пользователя
4. **Сохраняйте структуру** и форматирование (frontmatter, теги)
5. **Новые заметки** сначала в `00-Inbox/`, затем категоризировать
6. **Используйте кириллицу** в именах файлов (это русскоязычная база)
7. **Системные файлы** — на латинице (префикс `00-`)
