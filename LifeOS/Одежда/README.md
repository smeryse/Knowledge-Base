# 👔 Система управления одеждой

Автоматизированная система трекинга одежды, образов и стирки в Obsidian.

---

## 📁 Структура


## 🔄 Как это работает

### 1. Создание daily note
- При создании заметки через календарь, Templater спрашивает: **"Выбрать образ на сегодня?"**
- Введи `да` → появится список образов → выбери номер
- В заметку вставляется таблица образа + чекбокс "Образ надет"

### 2. Трекинг носки
- После того как носил образ весь день, **отметь чекбокс** `[x] Образ надет`
- Запусти команду Templater: `track-outfit.js` (или вручную обнови файлы)
- Счётчик `wears_count` у всех вещей образа увеличивается на 1

### 3. Автоматическая корзина
- Когда `wears_count >= wash_after_wears`, вещь помечается как `is_dirty: true`
- По цвету определяется корзина:
  - **Чёрная:** тёмные вещи (чёрные, тёмно-синие, бордовые)
  - **Белая:** светлые (белые, бежевые, светло-серые)
  - **Цветные:** остальные (стираются отдельно)

### 4. Оповещение о стирке
- В daily note показывается warning, если корзина >4.8 кг (80% от 6 кг)
- В `Корзина.md` видны прогресс-бары заполненности

### 5. После стирки
- Открой `Корзина.md` → раздел "После стирки"
- Запусти команду сброса для нужной корзины
- Счётчики `wears_count` обнуляются

## 🛠 Скрипты

### `wardrobe.js` — основная логика
```javascript
const wardrobe = require(tp.config.plugin_folder + "/scripts/wardrobe.js");

// Выбрать образ
await wardrobe.selectOutfit(tp);

// Трекать образ (обновить счётчики)
await wardrobe.trackOutfit(outfit, tp);

// Проверить корзины
const laundry = await wardrobe.checkLaundry(tp);

// Сгенерировать warning
const warning = wardrobe.generateLaundryWarning(laundry);

// Сбросить корзину
await wardrobe.resetLaundry('black', tp); // или 'white'
```

### `track-outfit.js` — команда для трекинга
Запускается вручную после носки образа.

### `convert-wardrobe.js` — конвертер базы
Запускать только если нужно пересоздать файлы в `items/` из таблицы.

## 📊 YAML формат файла вещи

```yaml
---
type: "футболка"
name: "Zolla Basic Collection Comfort Fit"
color: "темно-синяя"
color_group: black        # black | white | colored
size: "S"
weight: 180               # вес в граммах
weight_str: "~180 г"
wash_after_wears: 3       # лимит носки до стирки
wears_count: 0            # текущий счётчик
is_dirty: false           # готова к стирке
laundry_basket: null      # black | white | colored | null
---
```

## ⚙️ Настройки

### Лимит корзины
В `wardrobe.js` и `Корзина.md`:
```javascript
const LAUNDRY_LIMIT = 4800; // 4.8 кг = 80% от 6 кг
```

### Цветовые группы
- **black:** чёрные, тёмно-синие, бордовые, фиолетовые
- **white:** белые, бежевые, светло-серые
- **colored:** всё остальное

## 🐛 Решение проблем

### "Файл образов не найден"
Проверь что `LifeOS/Одежда/Образы.md` существует.

### "Не найдена вещь: ..."
Скрипт ищет по ключевым словам. Убедись что имя вещи в образе совпадает с именем в `items/`.

### Корзина не обновляется
Обнови `Корзина.md` (правый клик → Reload) или перезапусти Obsidian.

### Templater не спрашивает образ
Проверь настройки Templater:
- `Trigger Templater on new file creation` = включено
- `Template folder location` = `Templates`

## 📝 Важные заметки

- ✅ **Оригинальные файлы `Одежда база данных.md` и `Образы.md` НЕ изменяются**
- ✅ Файлы в `items/` создаются автоматически, можно пересоздать через `convert-wardrobe.js`
- ✅ Всё работает внутри Obsidian, не нужен внешний сервер
- ⚠️ Dataview плагин должен быть включён для работы `Корзина.md`
